HTML

Table Tags

Tag Name Description Reference
<table> Table The wrapper element for all HTML tables. MDN, W3Schools
<thead> Table Head The set of rows defining the column headers in a table. MDN, W3Schools
<tbody> Table Body This HTML element encapsulates a set of table rows, indicating that they comprise the body of the table. MDN, W3Schools
<tr> Table Row HTML element defines a row of cells in a table. The row's cells can then be established using a mix of <td> (data cell) and <th> (header cell) elements. MDN, W3Schools
<td> Table Data Defines a cell of a table that contains data. MDN, W3Schools
<tfoot> Table Foot Defines a set of rows summarizing the columns of the table. MDN, W3Schools
<colgroup> Table Column Group This HTML element defines a group of columns within a table. MDN, W3Schools
<col> Table Column HTML element defines a column within a table and is used for defining common semantics on all common cells. It is generally found within a <colgroup> element. MDN, W3Schools

Table attributes

Tag Attribute Description Reference
<td> colspan This attribute defines for how many columns the cell extends. MDN, W3Schools
rowspan This attribute defines for how many rows the cell extends. MDN, W3Schools
<colgroup> span This attribute defines the number of consecutive columns the <colgroup> element spans. MDN, W3Schools

HTML Form Tags

General reference: W3Schools

Tag Name Description Reference
<form> Form element The wrapper element for all HTML form elements. It represents a document section containing interactive controls for submitting information. MDN, W3Schools
<input> Input (Form Input) element This tag specifies an input field where the user can enter data. There are many different types, e.g. text (default), checkbox, submit, tel, password, etc. (see attributes). MDN, W3Schools
<select> Select element This element is used to create a drop-down list. The <option> tags inside the <select> element define the available options in the drop-down list. MDN, W3Schools
<datalist> Data List element This element specifies a list of pre-defined options for an <input> element. It is used to provide an "autocomplete" feature. MDN, W3Schools
<textarea> Textarea element This element defines a multi-line text input control. MDN, W3Schools

Form elements attributes

Tag Attribute Description Reference
<form> action This attribute specifies where to send the form-data when a form is submitted. MDN, W3Schools
method The HTTP method (GET, POST) to submit the form with. MDN, W3Schools
<input> type This attribute specifies the type of <input> element to display, such as:
  • text: a simple textbox
  • password: defines a password field (characters are masked)
  • number: defines a field for entering a number
  • range: defines a range control (like a slider control)
  • checkbox: defines a checkbox
  • radio: defines a radio button
  • submit: defines a submit button
 MDN, W3Schools
name Name of the form control. Submitted with the form as part of a name/value pair. Consider the name a required attribute (even though it's not). MDN, W3Schools
value value of an input element. Behaviour depends on the input type (see reference). MDN, W3Schools
required Makes input required. MDN, W3Schools
list The list attribute refers to a datalist element that contains pre-defined options for an input element. MDN, W3Schools
datalist rows Specifies the visible number of lines in a text area MDN, W3Schools
cols Specifies the visible width of a text area MDN, W3Schools
maxlength Specifies the maximum number of characters allowed in the text area MDN, W3Schools
placeholder A hint to the user of what can be entered in the control. MDN, W3Schools

Semenatic HTML Tags

General reference: W3Schools

Tag Name Description Reference
<header> Header Container for navigational links or introductory content. MDN, W3Schools
<nav> Navigation Container for major navigational links on the site. MDN, W3Schools

CSS

CSS selectors

General reference: MDN and W3Schools

Selector Description Example
* Universal selector: Selects all elements CSS: * {}
element Selects all specified HTML elements h1 {}
.class Selects all elements with the specified class HTML: <nav class="navigation"></nav>
CSS: .navigation {}
#id Selects the element with the specified id HTML: <div id="main"></div>
CSS: #main {}
[attribute] Selects all elements with a target attribute. Can be further specified to target attributes with specified values etc. HTML: <div class="main"></div>
CSS: [class] {}
.class1.class2 Selects all elements with both class1 and class2 set within its class attribute HTML: <div class="border highlight"></div>
CSS: .border.highlight {}
.class1 .class2 Descendant combinator: The " " (space) combinator selects all elements with class2 that is a descendant of an element with class1 HTML: <div class="container"><div class="border"></div></div>
CSS: .container .border {}
element, element The , selector is a grouping method that selects all the matching nodes. CSS: h1, h2, h3 {}
element > element Child combinator: The > combinator selects nodes that are direct children of the first element. HTML: <div class="container"><div class="border"></div></div>
CSS: .container > .border {}
element + element Adjacent sibling combinator: The + combinator matches the second element only if it immediately follows the first element. HTML: <div></div><p></p>
CSS: div + p {}
element ~ element General sibling combinator: The ~ combinator selects siblings. This means that the second element follows the first (though not necessarily immediately), and both share the same parent. HTML: <span>This is not red.</span> <p>Here is a paragraph.</p> <aside>Some aside content</code> <span>And here is a red span!</span>
CSS: p ~ span {color: red;}
: The : pseudo allow the selection of elements based on state information that is not contained in the document tree. Reference: MDN :hover, :active, :hover etc.
:: The :: pseudo represent entities that are not included in HTML. Reference: MDN ::after, ::before

Display Property in CSS

General reference: MDN, W3Schools, freeCodeCamp

Value Description Reference
inline The element generates one or more inline element boxes that do not generate line breaks before or after themselves. In normal flow, the next element will be on the same line if there is space
block The element generates a block element box, generating line breaks both before and after the element when in the normal flow.
inline-block The element generates a block element box that will be flowed with surrounding content as if it were a single inline box (behaving much like a replaced element would). W3Schools

Position Property in CSS

General reference: MDN, W3Schools

Value Description
static The element is positioned according to the normal flow of the document. This is the default value.
relative The element is positioned relative to its normal position, so "left:20px" adds 20 pixels to the element's LEFT position.
absolute The element is removed from the normal document flow, and no space is created for the element in the page layout. It is positioned relative to its closest positioned ancestor
fixed The element is removed from the normal document flow, and no space is created for the element in the page layout. The element is positioned relative to the browser window. Its final position is determined by the values of top, right, bottom, and left. Used to create a navigation bar that should always be visible.
sticky The element is positioned based on the user's scroll position.
